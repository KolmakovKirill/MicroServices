<h1>NotificationService</h1>

<p>
Проект выполнен в рамках студенческого обучения.
</p>

<p>
NotificationService — микросервисный сервис уведомлений, который принимает запросы
на отправку сообщений и доставляет их по различным каналам:
<b>Email</b>, <b>SMS</b>, <b>Push</b>, <b>Messenger</b>.
</p>

<h1>Архитектура</h1>

<p>
Сервис построен по микросервисной архитектуре с использованием брокера сообщений Kafka.
Шлюз (Gateway) принимает входящие HTTP-запросы и публикует события в Kafka,
а специализированные sender-сервисы обрабатывают их и выполняют фактическую отправку сообщений.
</p>

<h1>Репозиторий</h1>

<p><b>CommonShared</b> — общая кодовая база, используемая всеми сервисами (общие модели, утилиты, observability)</p>
<p><b>Gateway</b> — шлюз уведомлений, точка входа в приложение</p>
<p><b>EmailSender</b> — отправка email-уведомлений</p>
<p><b>SmsSender</b> — отправка SMS-уведомлений</p>
<p><b>PushSender</b> — отправка push-уведомлений</p>
<p><b>MessengerSender</b> — отправка сообщений через мессенджеры</p>

<h1>Observability и мониторинг</h1>

<p>
В проект добавлена полноценная система наблюдаемости (observability),
включающая сбор <b>логов</b>, <b>метрик</b> и <b>трейсов</b>.
</p>

<p>
Все сервисы используют <b>OpenTelemetry</b> для генерации телеметрии,
которая передаётся в централизованный <b>OpenTelemetry Collector</b>.
</p>

<ul>
  <li><b>Grafana</b> — единый UI для просмотра метрик, логов и трейсов</li>
  <li><b>Prometheus</b> — хранение и агрегация метрик</li>
  <li><b>Loki</b> — централизованное хранение логов</li>
  <li><b>Tempo</b> — хранение и анализ распределённых трейсов</li>
  <li><b>OpenTelemetry Collector</b> — приём, обработка и маршрутизация телеметрии</li>
</ul>

<p>
Это позволяет:
</p>
<ul>
  <li>анализировать производительность сервисов</li>
  <li>отслеживать путь одного запроса через несколько микросервисов</li>
  <li>коррелировать логи с конкретными трейcами</li>
  <li>выявлять ошибки и узкие места системы</li>
</ul>
